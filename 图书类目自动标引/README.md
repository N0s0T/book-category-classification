# 图书类目自动标引系统

一个基于机器学习的图书自动分类系统，可以根据图书的标题、关键词和摘要，自动判断图书属于哪个类别。

## 项目简介

这个项目是一个完整的图书分类系统。简单来说，就是让计算机学会给图书分类。

比如，你给它一本书的信息（标题、关键词、摘要），它能告诉你这本书是"计算机科学"、"文学"还是"历史"等类别。

### 系统能做什么

- 自动处理图书数据（分词、清洗、合并文本）
- 训练机器学习模型进行分类
- 对比多种模型，找出效果最好的
- 自动调整模型参数，提高准确率
- 生成可视化图表，方便分析结果

## 实验结果

我们测试了三种主流的机器学习模型：

| 模型 | 准确率 | 说明 |
|------|--------|------|
| SVM（支持向量机） | 31.57% | 效果最好 |
| 逻辑回归 | 30.12% | 效果次之 |
| 朴素贝叶斯 | 26.84% | 效果一般 |

数据规模：训练数据8155条，验证数据2965条，共200多个类别。

**为什么准确率看起来不高？**

因为类别太多了（200多个），这是一个很难的分类任务。如果随机猜，准确率只有0.5%左右，所以31%已经是很好的结果了。

## 代码详细说明

### 12.py - 数据预处理

**做什么：** 把原始的图书数据整理成模型能用的格式

**具体步骤：**
1. 读取原始CSV文件
2. 对中文文本进行分词（用jieba工具）
3. 去掉无意义的停用词（如"的"、"是"、"在"等）
4. 把标题、关键词、摘要合并成一段完整的文本
5. 保存处理后的数据

**为什么需要预处理：** 原始数据是给人看的，机器学习模型需要特定格式的数据才能处理。

### 13.py - 模型训练

**做什么：** 训练一个图书分类模型

**具体步骤：**
1. 加载预处理后的数据
2. 把文本转换成数字特征（用TF-IDF技术）
3. 训练SVM模型
4. 在验证集上测试效果
5. 输出分类报告

**关键技术：TF-IDF**

TF-IDF是一种把文本转换成数字的方法。简单说：
- TF（词频）：一个词在文档中出现的次数
- IDF（逆文档频率）：一个词在所有文档中出现的频率

一个词如果在某个文档中出现很多次，但在其他文档中很少出现，那这个词就很重要，TF-IDF值就高。

### 14.py - 模型对比与调优

**做什么：** 对比多种模型，找出最好的，并优化参数

**具体步骤：**
1. 训练4种不同的模型
2. 在验证集上对比效果
3. 分析为什么某些模型效果好，某些效果差
4. 用网格搜索自动找最优参数
5. 输出完整的对比报告

**为什么要对比多种模型：** 不同模型适合不同的问题，不试一下不知道哪个好。

**什么是网格搜索：** 就是一种自动调参的方法。比如SVM有个参数C，我们不知道设多少好，就让计算机把可能的值都试一遍（0.01, 0.1, 0.5, 1.0...），然后选效果最好的。

### 15.py - 可视化分析

**做什么：** 生成各种图表，直观展示模型效果

**具体步骤：**
1. 绘制混淆矩阵（看哪些类别容易混淆）
2. 绘制ROC曲线（评估模型区分能力）
3. 绘制特征重要性图（看哪些词最重要）
4. 绘制模型对比图（对比不同模型）
5. 绘制调参过程图（看参数如何影响效果）

**为什么要可视化：** 图表比数字更直观，更容易发现问题和规律。

## 可视化图表解读

### 1. 混淆矩阵

这是一个方格图，用来展示分类结果。

- 对角线上的格子：预测正确的样本
- 非对角线的格子：预测错误的样本
- 颜色越深：样本越多

**怎么看：** 如果对角线颜色很深，其他地方颜色很浅，说明模型效果很好。

### 2. TOP10混淆矩阵

只展示最容易搞混的10个类别。

**怎么看：** 看非对角线的格子，数字越大说明这两个类别越容易混淆。

### 3. ROC曲线

用来评估模型区分正例和负例的能力。

- 横轴：假正例率（把负例错判为正例的比例）
- 纵轴：真正例率（正确判断正例的比例）
- AUC值：曲线下面积，越接近1越好

**怎么看：** 曲线越靠近左上角，AUC值越接近1，模型效果越好。

### 4. 特征重要性图

展示对分类贡献最大的词。

**怎么看：** 排在前面的词最重要。比如如果"计算机"、"算法"、"编程"排在前几位，说明这些词对判断"计算机科学"类别很重要。

### 5. 模型对比图

用柱状图对比不同模型的准确率。

**怎么看：** 柱子越高，模型效果越好。

### 6. 超参数调优图

展示不同参数值对模型效果的影响。

**怎么看：** 曲线的最高点就是最优参数值。

## 技术要点

### TF-IDF参数说明

```python
TfidfVectorizer(
    max_features=8000,    # 最多保留8000个词
    ngram_range=(1, 2),   # 用单个词和两个词的组合
    min_df=2,             # 至少在2个文档中出现才保留
    max_df=0.95,          # 在超过95%文档中出现的词不保留
    sublinear_tf=True     # 用对数变换，让数值更稳定
)
```

### SVM参数说明

```python
LinearSVC(
    C=0.1,         # 正则化参数，越小越保守（防止过拟合）
    max_iter=2000  # 最多迭代2000次
)
```

**什么是过拟合：** 模型在训练数据上表现很好，但在新数据上表现很差。就像学生死记硬背答案，考试遇到新题就不会了。

## 实验结论

### 1. 哪个模型最好？

SVM（支持向量机）效果最好，准确率31.57%。

**为什么SVM效果好：**
- SVM擅长处理高维数据（我们有很多特征）
- SVM通过最大化分类间隔，泛化能力强
- 正则化参数C=0.1有效防止过拟合

### 2. 为什么准确率不高？

- 类别太多（200多个），分类难度大
- 文本信息有限（只有标题、关键词、摘要）
- 有些类别样本很少，模型学不好

### 3. 如何改进？

**短期改进：**
- 增加训练数据
- 调整TF-IDF参数
- 尝试更多模型

**长期改进：**
- 使用BERT等预训练模型
- 加入图书正文内容
- 采用层次分类（先分大类，再分小类）

## 常见问题

**Q: 运行时报错找不到文件？**

A: 确保数据文件在正确的位置。数据应该放在"第3章 数据集"文件夹中。

**Q: 中文显示乱码？**

A: 代码中已经设置了中文字体，如果还是乱码，可能需要安装SimHei字体。

**Q: 运行很慢？**

A: 这是正常的，训练模型和处理文本需要时间。如果太慢，可以减少特征数量（把max_features从8000改小）。

**Q: 准确率为什么这么低？**

A: 200多个类别的分类任务本身就很难。可以尝试减少类别数量，或者使用更先进的模型（如BERT）。
