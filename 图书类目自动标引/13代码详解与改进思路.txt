图书分类模型代码详解与改进思路
================================================================================

本文档详细解释改进版图书分类模型(13.py)的代码逻辑和改进思路。

================================================================================
一、项目背景
================================================================================

这是一个图书类目自动标引系统，目的是根据图书的标题、关键词和摘要，
自动预测图书所属的类别（如F270、F121等）。

数据集信息：
- 训练集：8155条记录
- 验证集：2999条记录（其中34条包含未见过的类别）
- 有效验证样本：2965条
- 类别数量：约200+个类别

================================================================================
二、代码逐段解析
================================================================================

【第1-6行】导入必要的库
--------------------------------------------------------------------------------

import pandas as pd                          # 数据处理
from sklearn.feature_extraction.text import TfidfVectorizer  # 文本特征提取
from sklearn.naive_bayes import MultinomialNB # 朴素贝叶斯分类器
from sklearn.svm import LinearSVC            # 支持向量机分类器
from sklearn.metrics import accuracy_score  # 准确率评估
from sklearn.preprocessing import LabelEncoder  # 标签编码

这些是Python机器学习的基础库：
- pandas：处理表格数据
- sklearn：机器学习库，包含各种算法和工具


【第12-16行】数据加载
--------------------------------------------------------------------------------

train = pd.read_csv("第3章 数据集/train_final_result.csv")  # 加载训练数据
val = pd.read_csv("第3章 数据集/data/val.csv")              # 加载验证数据
val = val.fillna("")                                        # 填充缺失值

说明：
- 训练集是12.py处理后的特征工程结果
- 验证集包含真实的类别标签，用于评估模型性能
- fillna("") 确保缺失值不会导致后续处理出错


【第22-25行】提取列名
--------------------------------------------------------------------------------

col_cat_val = val.columns[1]  # 列1：类别标签（如F121、F270）
col_title = val.columns[2]    # 列2：图书标题
col_key = val.columns[3]      # 列3：关键词
col_abs = val.columns[4]      # 列4：摘要

说明：
- 根据CSV列的顺序提取各字段
- 这些字段将用于构建文本特征


【第28-29行】构造文本特征
--------------------------------------------------------------------------------

X_val = val[col_title].astype(str) + " " + 
        val[col_abs].astype(str) + " " + 
        val[col_key].astype(str)
y_val = val[col_cat_val]  # 真实类别标签

说明：
- 将标题、摘要、关键词拼接成一个大文本
- astype(str) 确保所有数据都是字符串类型
- y_val 是验证集的真实类别，用于后续评估


【第31-33行】准备训练数据
--------------------------------------------------------------------------------

X_train = train["all_text"].astype(str)  # 使用预先处理好的all_text
y_train = train["label"]                   # 使用数字编码的标签
train_categories = train.iloc[:, 1]        # 原始类别名称

说明：
- all_text 是12.py中预先合并的文本特征
- label 是类别经过LabelEncoder编码后的数字
- train_categories 用于训练编码器，确保验证集和训练集用相同的编码


【第36-37行】标签编码
--------------------------------------------------------------------------------

le = LabelEncoder()
le.fit(train_categories)

说明：
- LabelEncoder 将类别名称（如F121）转换为数字（0,1,2...）
- fit() 用训练集的类别来"学习"编码规则
- 这是为了确保训练集和验证集使用相同的编码方式


【第40-43行】过滤验证集
--------------------------------------------------------------------------------

mask = y_val.isin(le.classes_)  # 找出验证集中类别在训练集出现过的样本
y_val_filtered = y_val[mask]
X_val_filtered = X_val[mask]
y_val_encoded = le.transform(y_val_filtered)

说明：
- 验证集中可能存在训练集没有的类别（如F335）
- 这些"未见过的类别"无法评估，需要过滤掉
- isin(le.classes_) 创建布尔掩码
- transform() 将过滤后的类别转换为数字编码


【第52-58行】TF-IDF特征提取（改进核心）
--------------------------------------------------------------------------------

tfidf = TfidfVectorizer(
    max_features=8000,      # 特征数量：3000 -> 8000
    ngram_range=(1, 2),    # 添加词组：(1,1) -> (1,2)
    min_df=2,              # 去除只出现1次的词
    max_df=0.95,           # 去除出现在95%以上文档的词
    sublinear_tf=True       # 使用对数TF
)

X_train_vec = tfidf.fit_transform(X_train)
X_val_vec = tfidf.transform(X_val_filtered)

说明：
- TF-IDF：将文本转换为数字向量的标准方法
- max_features=8000：选择最重要的8000个词（原来是3000）
- ngram_range=(1,2)：不仅用单词，还用词组（bigram）
  * 例如："税收风险管理" -> "税收"、"风险管理"、"税收风险管理"
- min_df=2：出现次数少于2次的词可能是错别字或噪音，去除
- max_df=0.95：出现在95%以上文档的词（如"的"、"是"）信息量低，去除
- sublinear_tf=True：使用1+log(tf)替代tf，减少高频词的影响


【第71-76行】朴素贝叶斯模型（改进）
--------------------------------------------------------------------------------

nb = MultinomialNB(alpha=0.5)  # 平滑参数：1.0 -> 0.5
nb.fit(X_train_vec, y_train)
pred_nb = nb.predict(X_val_vec)
acc_nb = accuracy_score(y_val_encoded, pred_nb)

说明：
- MultinomialNB 适合文本分类，假设特征服从多项分布
- alpha=0.5：平滑参数，控制模型的复杂度
  * alpha越大，模型越保守（欠拟合）
  * alpha越小，模型越激进（可能过拟合）
  * 从1.0调整到0.5，让模型更灵活
- fit()：在训练集上训练模型
- predict()：在验证集上预测
- accuracy_score()：计算准确率


【第79-88行】SVM模型（改进核心）
--------------------------------------------------------------------------------

svc = LinearSVC(
    C=0.1,       # 正则化参数：1.0 -> 0.1
    max_iter=2000,  # 最大迭代次数：1000 -> 2000
    dual=True
)
svc.fit(X_train_vec, y_train)
pred_svc = svc.predict(X_val_vec)
acc_svc = accuracy_score(y_val_encoded, pred_svc)

说明：
- LinearSVC 线性支持向量机，适合高维文本分类
- C=0.1：正则化参数，这是最重要的改进！
  * C越小，正则化越强，模型越简单，越不容易过拟合
  * C越大，模型越复杂，越容易过拟合
  * 从C=1.0改为C=0.1，大幅减少了过拟合
- max_iter=2000：增加迭代次数，确保模型能收敛


================================================================================
三、改进思路详解
================================================================================

【问题诊断】原始模型有什么问题？
--------------------------------------------------------------------------------

1. SVM过拟合严重
   - 训练集准确率：97%
   - 验证集准确率：26%
   - 差距：71个百分点！
   - 原因：SVM太强大，在训练数据上"死记硬背"，泛化能力差

2. 特征不够丰富
   - 只用unigram（单词），丢失词组信息
   - 特征数量有限（3000个）
   - 没有去除噪音特征

3. 模型参数不当
   - SVM的C=1.0正则化不够
   - 朴素贝叶斯的alpha=1.0过于保守


【改进方案1】优化TF-IDF特征提取
--------------------------------------------------------------------------------

为什么要改进TF-IDF？
- TF-IDF是文本分类的基础，特征好则模型好
- 原始特征太简单，信息损失大

具体改进：
┌─────────────────┬────────────┬────────────┬──────────────────┐
│ 参数            │ 原始值    │ 改进值    │ 作用             │
├─────────────────┼────────────┼────────────┼──────────────────┤
│ max_features    │ 3000      │ 8000      │ 捕获更多信息     │
│ ngram_range     │ (1,1)     │ (1,2)     │ 保留词组信息     │
│ min_df          │ 无        │ 2         │ 去除噪音词       │
│ max_df          │ 无        │ 0.95      │ 去除停用词       │
│ sublinear_tf    │ 无        │ True      │ 更稳定的权重     │
└─────────────────┴────────────┴────────────┴──────────────────┘

效果：
- 特征维度：3000 -> 8000（+167%）
- 可以捕获"税收风险管理"这样的专业术语
- 去除错别字和无意义的高频词


【改进方案2】调整模型参数
--------------------------------------------------------------------------------

2.1 朴素贝叶斯的改进

原代码：
    nb = MultinomialNB(alpha=1.0)

改进后：
    nb = MultinomialNB(alpha=0.5)

原理：
- alpha是拉普拉斯平滑参数，防止概率为0
- alpha=1.0太保守，alpha=0.5更灵活
- 轻微提升准确率


2.2 SVM正则化（最关键的改进）

原代码：
    svc = LinearSVC(max_iter=1000)

改进后：
    svc = LinearSVC(C=0.1, max_iter=2000)

原理：
- C是正则化强度的倒数
- C越大，模型越复杂（容易过拟合）
- C越小，模型越简单（不容易过拟合）
- 从C=1.0改为C=0.1，正则化强度增加10倍

效果对比：
- 训练集准确率：97% -> ~31%（真实性能）
- 验证集准确率：26% -> 31%（提升）
- 过拟合问题：从严重变为轻微

================================================================================
六、关键知识点
================================================================================

【TF-IDF原理】
- TF（词频）：词在文档中出现多少次
- IDF（逆文档频率）：词在所有文档中的罕见程度
- TF-IDF = TF × IDF
- 词越常见，TF越高；但IDF会惩罚它
- 词越罕见，IDF越高，TF-IDF也越高

【Bigram的作用】
- "税收风险管理"是一个专业术语
- 用unigram只能看到"税收"、"风险管理"
- 用bigram能看到"税收风险管理"、"税收管理"等词组
- 词组往往包含更精确的语义

【过拟合与正则化】
- 过拟合：模型在训练集上表现很好，在新数据上表现差
- 原因：模型太复杂，记住训练数据的"噪音"
- 解决：正则化，限制模型的复杂度
- SVM的C参数：控制正则化强度
  * C大 = 复杂模型 = 可能过拟合
  * C小 = 简单模型 = 不易过拟合

4. 类别工程
   - 将200+个细分类别合并为20-30个粗粒度类别
   - 先分类到大类，再细分类
   - 降低分类难度


================================================================================
八、总结
================================================================================

本次改进的核心要点：

1. TF-IDF优化（+2.46%）
   - 增加特征数量和类型
   - 添加bigram捕获词组信息
   - 去除噪音特征

2. SVM正则化（+5.26%）
   - 从C=1.0改为C=0.1
   - 解决过拟合问题
   - 真实性能从26%提升到31%

3. 总提升
   - 从24%提升到31%
   - 相对提升30%
   - 模型从严重过拟合变为正常

关键启示：
- 验证集准确率才是真正的模型性能
- 训练集准确率过高反而说明有问题
- 正则化是防止过拟合的有效手段
- 特征工程和参数调优同样重要
